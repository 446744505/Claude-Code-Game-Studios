# Hook：post-sprint-retrospective（冲刺结束后复盘）

## 触发时机

每个冲刺结束时手动触发（通常由制作人 agent 或人类开发者调用）。

## 目的

通过分析冲刺数据自动生成复盘起点：计划与完成情况、速率变化、缺陷趋势与常见阻塞项。这不是 Git hook，而是通过 `producer` agent 调用的工作流 hook。

## 实现方式

这是工作流 hook，不是 Git hook。通过以下方式调用：

```
@producer 为 Sprint [N] 生成冲刺复盘
```

制作人 agent 应：

1. **读取冲刺计划**：自 `production/sprints/sprint-[N].md`
2. **计算指标**：
   - 计划任务数 vs 完成任务数
   - 计划故事点 vs 完成故事点（若使用）
   - 自上一冲刺带入的未完项
   - 冲刺中途新增任务
   - 平均任务完成时间
3. **分析模式**：
   - 最常见阻塞项
   - 哪个 agent/领域未完成工作最多
   - 哪些估算偏差最大
4. **生成复盘文档**：

```markdown
# 冲刺 [N] 复盘

## 指标
| 指标 | 数值 |
|------|------|
| 计划任务数 | [N] |
| 完成任务数 | [N] |
| 完成率 | [X%] |
| 自上一冲刺带入 | [N] |
| 中途新增任务 | [N] |
| 发现缺陷数 | [N] |
| 修复缺陷数 | [N] |

## 速率趋势
[冲刺 N-2]：[X] | [冲刺 N-1]：[Y] | [冲刺 N]：[Z]
趋势：[改善 / 稳定 / 下滑]

## 做得好的地方
- [自动识别：早于估算完成的任务]
- [主持人补充团队观察]

## 做得不好的地方
- [自动识别：被顺延或砍掉的任务]
- [自动识别：估算显著超支的领域]
- [主持人补充团队观察]

## 阻塞项
| 阻塞项 | 出现频次 | 解决耗时 | 预防 |
|--------|----------|----------|------|

## 下一冲刺行动项
| # | 行动项 | 负责人 | 优先级 |
|---|--------|--------|--------|

## 估算准确度
| 领域 | 平均计划 | 平均实际 | 准确度 |
|------|----------|----------|--------|
```

5. **保存**至 `production/sprints/sprint-[N]-retro.md`
