当本技能被调用时：

> **仅限显式调用**：本技能仅应在用户用 `/release-checklist` 明确要求时运行。请勿仅因上下文匹配而自动调用。

1. **读取参数**，确定目标平台（`pc`、`console`、`mobile` 或 `all`）。若未指定平台，默认为 `all`。

2. **阅读 CLAUDE.md**，获取项目上下文、版本信息与目标平台。

3. **从 `production/milestones/` 读取当前里程碑**，了解本版本应包含的功能与内容。

4. **扫描代码库**中的未决问题：
   - 统计 `TODO` 注释数量
   - 统计 `FIXME` 注释数量
   - 统计 `HACK` 注释数量
   - 记录其位置与严重程度

5. **检查测试结果**：若存在测试输出目录或 CI 日志，则查阅其中结果。

6. **生成发布清单**：

```markdown
## 发布清单：[Version] -- [Platform]
生成时间：[Date]

### 代码库健康度
- TODO 数量：[N]（若很多则列出前 5 条）
- FIXME 数量：[N]（全部列出 — 这些可能是阻塞项）
- HACK 数量：[N]（全部列出 — 这些需要评审）

### 构建验证
- [ ] 所有目标平台干净构建均成功
- [ ] 无编译器警告（零警告策略）
- [ ] 所有资源已包含且加载正确
- [ ] 构建体积在预算内（[target size]）
- [ ] 构建版本号已正确设置（[version]）
- [ ] 构建可从带标签的提交复现

### 质量门禁
- [ ] 零 S1（严重）缺陷
- [ ] 零 S2（主要）缺陷 — 或已有制作人批准的书面例外
- [ ] 所有关键路径功能已由 QA 测试并签字
- [ ] 性能在预算内：
  - [ ] 最低配置硬件上达到目标 FPS
  - [ ] 内存使用在预算内
  - [ ] 加载时间在预算内
  - [ ] 长时间游玩无内存泄漏
- [ ] 相对上一构建无回归
- [ ] 浸泡测试通过（连续游玩 4+ 小时）

### 内容完备
- [ ] 所有占位资源已替换为正式版
- [ ] 内容文件中的 TODO/FIXME 已解决或已文档化
- [ ] 所有面向玩家的文案已校对
- [ ] 所有文案已具备本地化条件（无硬编码字符串）
- [ ] 音频混音已定稿并获批
- [ ] 制作人员名单完整且准确
```

7. **根据参数追加平台专属小节**：

`pc`：
```markdown
### 平台要求：PC
- [ ] 最低与推荐配置已验证并文档化
- [ ] 键鼠操作完全可用
- [ ] 手柄支持已测试（Xbox、PlayStation、通用）
- [ ] 分辨率缩放已测试（1080p、1440p、4K、超宽屏）
- [ ] 窗口化、无边框与全屏模式均正常
- [ ] 图形设置可正确保存与加载
- [ ] Steam/Epic/GOG SDK 已集成并测试
- [ ] 成就功能正常
- [ ] 云存档功能正常
- [ ] 若目标包含 Steam Deck，已验证兼容性
```

`console`：
```markdown
### 平台要求：主机
- [ ] TRC/TCR/Lotcheck 要求清单已完成
- [ ] 平台专属手柄提示显示正确
- [ ] 挂起/恢复工作正常
- [ ] 用户切换处理正确
- [ ] 网络断开时能优雅降级
- [ ] 存储空间已满场景已处理
- [ ] 家长控制已遵守
- [ ] 平台专属成就/奖杯集成已测试
- [ ] 第一方认证提交材料已准备
```

`mobile`：
```markdown
### 平台要求：移动端
- [ ] 应用商店准则合规已核实
- [ ] 所需设备权限均有正当理由并已文档化
- [ ] 隐私政策链接正确且内容准确
- [ ] 数据安全/营养标签等信息已填写
- [ ] 触控在多种屏幕尺寸上已测试
- [ ] 电池消耗在可接受范围内
- [ ] 后台行为正确（暂停、恢复、终止）
- [ ] 推送通知权限处理正确
- [ ] 应用内购买流程已测试（如适用）
- [ ] 应用体积在商店限制内
```

8. **追加商店与上线小节**：

```markdown
### 商店 / 分发
- [ ] 商店页元数据完整且已校对
  - [ ] 短描述
  - [ ] 长描述
  - [ ] 功能列表
  - [ ] 系统需求（PC）
- [ ] 截图已更新且满足各平台分辨率要求
- [ ] 预告片已更新
- [ ] 主视觉与胶囊图已更新
- [ ] 年龄分级已取得并配置：
  - [ ] ESRB
  - [ ] PEGI
  - [ ] 其他地区所需分级
- [ ] 法律声明、EULA 与隐私政策已就位
- [ ] 第三方许可署名完整
- [ ] 各区域定价已配置

### 上线就绪
- [ ] Analytics / 遥测已验证且能收到数据
- [ ] 崩溃上报已配置且可访问看板
- [ ] 首日补丁已准备并测试（如需要）
- [ ] 上线后前 72 小时值班表已排定
- [ ] 社区上线公告草稿已准备
- [ ] 媒体/主播兑换码已准备分发
- [ ] 支持团队已了解已知问题与 FAQ
- [ ] 回滚方案已文档化（若上线后发现严重问题）

### 放行 / 不放行：[READY / NOT READY]

**理由：**
[就绪评估摘要。列出上线前必须解决的阻塞项。若为 NOT READY，列出需处理的具体事项及预估所需时间。]

**所需签字：**
- [ ] QA 负责人
- [ ] 技术总监
- [ ] 制作人
- [ ] 创意总监
```

9. **将清单保存到** `production/releases/release-checklist-[version].md`；若目录不存在则创建。

10. **向用户输出摘要**，包括：清单条目总数、已知阻塞项数量（FIXME/HACK 计数、已知缺陷），以及文件路径。
