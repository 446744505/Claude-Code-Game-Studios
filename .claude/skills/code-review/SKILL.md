本技能被调用时：

1. **完整阅读目标文件**。

2. **阅读 CLAUDE.md**，以了解项目编码规范。

3. **识别系统类别**（引擎、玩法、AI、网络、UI、工具），
   并按类别应用相应标准。

4. **对照编码规范评估**：
   - [ ] 公共方法与类具备文档注释
   - [ ] 每个方法的圈复杂度低于 10
   - [ ] 无方法超过 40 行（不含数据声明）
   - [ ] 依赖为注入式（游戏状态不使用静态单例）
   - [ ] 配置值从数据文件加载
   - [ ] 系统对外暴露接口（不依赖具体实现类）

5. **检查架构符合度**：
   - [ ] 依赖方向正确（引擎 ← 玩法，不可反向）
   - [ ] 模块间无循环依赖
   - [ ] 分层清晰（UI 不持有游戏状态）
   - [ ] 跨系统通信使用事件/信号
   - [ ] 与代码库中既有模式一致

6. **检查 SOLID 符合度**：
   - [ ] 单一职责：每个类仅有一个变更理由
   - [ ] 开闭原则：可扩展而无需修改既有代码
   - [ ] 里氏替换：子类型可替换基类型
   - [ ] 接口隔离：无臃肿接口
   - [ ] 依赖倒置：依赖抽象，不依赖具体实现

7. **检查常见游戏开发问题**：
   - [ ] 与帧率无关（正确使用 delta time）
   - [ ] 热路径（如 update 循环）无分配
   - [ ] 妥善处理 null/空状态
   - [ ] 在需要处保证线程安全
   - [ ] 资源正确释放（无泄漏）

8. **按以下格式输出审阅结果**：

```
## 代码审阅：[文件/系统名称]

### 规范符合：[X/6 项通过]
[列出未通过项并附行号引用]

### 架构：[干净 / 小问题 / 发现违规]
[列出具体架构关切]

### SOLID：[符合 / 发现问题]
[列出具体违反项]

### 游戏相关关切
[列出游戏开发特有的问题]

### 正面观察
[做得好的地方——本段必须包含]

### 必须修改
[批准前必须修复的项]

### 建议
[可选的改进]

### 结论：[通过 / 通过（含建议）/ 需修改]
```
